`module_bindings/` is reserved for Rust bindings generated by the StateStore CLI.

The real generated bindings live under `src/module_bindings/generated/`.

This workspace uses `build.rs` to:

1. build the `state_store/` module crate to wasm
2. run `state-module generate --lang rust --bin-path <module.wasm> --out-dir src/module_bindings/generated --yes`

If the CLI is unavailable or generation fails, the last generated bindings remain in place so the
workspace can still compile. On a fresh checkout without generated files, `build.rs` writes a tiny
placeholder `generated/mod.rs`.
